# Demo Flow Verification Checklist

This document outlines the step-by-step verification for each role in the FamilyFirst application.

## 1. Parent Role Flow
- [ ] **Login**: Access via Demo Login -> Parent.
- [ ] **Dashboard**: Verify greeting, stats (Streak, Pending Tasks, Shop), and Quick Actions.
- [ ] **Child Detail**: Click on a child card, verify attendance trend and task list.
- [ ] **Verification**: Go to 'Verify Tasks', approve/reject a task.
- [ ] **Reward Shop**: Browse rewards, verify 'Approve' flow.
- [ ] **Routine Builder**: Navigate to a child's routine, add/edit a task.
- [ ] **Admin Panel**: Access Family Admin, toggle module visibility.
- [ ] **Subscription**: Verify plan details and upgrade flow.

## 2. Child Role Flow
- [ ] **Login**: Access via Demo Login -> Child.
- [ ] **My Day**: Verify task list (Morning, Afternoon, Evening).
- [ ] **Task Submission**: Click a task, verify 'Submit' flow with proof.
- [ ] **Rewards**: Check coin balance, browse available rewards.
- [ ] **Scores**: View performance charts and badges.
- [ ] **Family**: View family members and elder appreciations.

## 3. Teacher Role Flow
- [ ] **Login**: Access via Demo Login -> Teacher.
- [ ] **Attendance**: Select a class, mark students as Present/Absent/Late.
- [ ] **Feedback**: Select a student, submit a 'Star' or 'Note'.
- [ ] **History**: View past feedback submissions.

## 4. Elder Role Flow
- [ ] **Login**: Access via Demo Login -> Elder.
- [ ] **Updates**: View latest family achievements and photos.
- [ ] **Appreciation**: Send a 'Heart' or 'Blessing' to a grandchild.

## 5. Super Admin Flow
- [ ] **Analytics**: View DAU/WAU trends and feature usage heatmap.
- [ ] **Support**: View and reply to support tickets.
- [ ] **Content**: Edit onboarding slides or FAQs.

## 6. Global Features
- [ ] **Notifications**: Verify history list and preference toggles.
- [ ] **Calendar**: View monthly events, create a new family event.
- [ ] **Offline Mode**: Disable internet, verify amber banner and cached data.
- [ ] **Error Handling**: Verify retry buttons on failed loads.
